# 周计划增删改查页面实施计划

## 1. Summary
- 目标：在现有错题 Web 应用中新增「周计划」页面，支持按周（周一-周日）管理多条计划项，并提供增删改查与“完成/未完成”勾选。
- 要求：页面美观、整齐、易用；第一版计划内容以文本为主。

## 2. Current State Analysis
- 前端为单页 React，通过 `tab` 状态在同一个组件内切换页面：[App.jsx](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/client/App.jsx)
- 现有样式为自定义 CSS，已经具备 `card / btn / form-grid` 等基础 UI 组件样式：[App.css](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/client/App.css)
- 后端为 Express + SQLite，启动时执行建表逻辑：[schema.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/db/schema.js)，并在 [app.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/app.js) 中挂载 API 路由。
- 现有 API 主要覆盖错题、OCR、导出等；前端 API 封装集中在：[clientApi.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/client/clientApi.js)

## 3. Assumptions & Decisions
- 周范围定义：周一-周日（选择任意日期后归到当周周一）。
- 数据结构：一周可包含多条计划项，每条计划包含：
  - `week_start_date`（该周周一，`YYYY-MM-DD`）
  - `content`（文本内容）
  - `is_done`（是否完成，默认未完成）
- 认证：仍不做登录（沿用当前模式）。

## 4. Proposed Changes

### 4.1 数据库（SQLite）新增表
- 修改文件：
  - [schema.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/db/schema.js)
- 新增表 `weekly_plan_items`：
  - `id TEXT PRIMARY KEY`
  - `week_start_date TEXT NOT NULL`
  - `content TEXT NOT NULL`
  - `is_done INTEGER NOT NULL DEFAULT 0`（0/1）
  - `created_at TEXT NOT NULL`
  - `updated_at TEXT NOT NULL`
- 新增索引：
  - `idx_weekly_plan_week_start_date`（按周查询）
- 迁移策略：
  - 采用 `CREATE TABLE IF NOT EXISTS` 增量建表；不影响现有错题数据。

### 4.2 后端 API：周计划 CRUD
- 新增文件：
  - `src/server/routes/weeklyPlanRoutes.js`
- 修改文件：
  - [app.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/app.js)（新增路由挂载）
- 路由设计（JSON）：
  - `GET /api/weekly-plans?weekStart=YYYY-MM-DD`
    - 返回该周所有计划项（按 `created_at` 或 `updated_at` 排序）
  - `POST /api/weekly-plans`
    - body: `{ weekStart, content }`
    - 返回 `{ id }`
  - `PUT /api/weekly-plans/:id`
    - body: `{ content?, is_done? }`（允许只更新勾选状态）
    - 返回 `{ id }`
  - `DELETE /api/weekly-plans/:id`
    - 返回 `{ id }`
- 服务端校验：
  - `weekStart` 必须为 `YYYY-MM-DD` 且是周一（若不是，服务端可自动归一化到周一，或直接返回 400；本期采用“自动归一化”以减少用户出错）
  - `content` 不能为空（trim 后）
  - `is_done` 仅允许 0/1（或 boolean 映射）
