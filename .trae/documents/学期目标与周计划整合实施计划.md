# 学期目标与周计划整合实施计划

## 1. Summary
- 目标：新增「学期目标」能力，并在「周计划」页面顶部固定展示（提醒作用），同时支持“按学期留档”。
- 学期目标默认内容（可编辑）：
  - 语文：期末考试 90 分以上
  - 数学：期末考试 95 分以上
  - 英语：期末考试 95 分以上

## 2. Current State Analysis
- 前端：单页 React，通过 `tab` 切换页面；「周计划」为第三个 Tab，并已具备卡片化基础样式：[App.jsx](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/client/App.jsx)、[App.css](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/client/App.css)
- 后端：Express + SQLite，启动时执行建表逻辑：[schema.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/db/schema.js)，路由在 [app.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/app.js) 挂载。
- 目前已有 `weekly_plan_items` 表与 `/api/weekly-plans` CRUD。

## 3. Assumptions & Decisions
- 展示方式：放在「周计划」页面顶部固定展示一张“学期目标卡片”（提醒为主）。
- 留档方式：按学期留档（可切换查看历史学期目标）。
- 学期标识：采用 `semester_name`（字符串）作为学期名称，例如 `2026春`、`2025-2026下`；不强制固定格式，但必填且唯一。
- 学科固定：语文/数学/英语；目标为数字分数（整数）。

## 4. Proposed Changes

### 4.1 数据库新增学期目标表
- 修改文件：
  - [schema.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/db/schema.js)
- 新增表 `semester_goals`（一行代表一个学期）：
  - `id TEXT PRIMARY KEY`
  - `semester_name TEXT NOT NULL UNIQUE`
  - `chinese_target INTEGER NOT NULL`
  - `math_target INTEGER NOT NULL`
  - `english_target INTEGER NOT NULL`
  - `created_at TEXT NOT NULL`
  - `updated_at TEXT NOT NULL`
- 索引/约束：
  - `UNIQUE(semester_name)` 防止重复学期名

### 4.2 后端 API：学期目标 CRUD + 列表
- 新增文件：
  - `src/server/routes/semesterGoalRoutes.js`
- 修改文件：
  - [app.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/app.js)（挂载 `/api/semester-goals`）
- API 设计：
  - `GET /api/semester-goals`
    - 返回学期列表（按 `created_at DESC`）用于下拉选择：`[{ semester_name, chinese_target, math_target, english_target, updated_at }]`
  - `GET /api/semester-goals/:semesterName`
    - 返回单个学期目标详情
  - `POST /api/semester-goals`
    - body: `{ semester_name, chinese_target, math_target, english_target }`
    - 校验：学期名非空；目标分数为 0~100 的整数；学期名唯一
  - `PUT /api/semester-goals/:semesterName`
    - body: `{ chinese_target?, math_target?, english_target? }`（更新分数）
  - `DELETE /api/semester-goals/:semesterName`
    - 用于清理历史学期（可选；若你希望保留历史不允许删除，则不实现该接口）
- 兼容策略：
  - 若表为空：服务端可在首次启动或首次查询时自动创建一个默认学期（例如 `当前学期`），并写入默认目标；或由前端引导创建。为减少首次使用成本，本期采用“前端提示 + 一键创建默认”。
