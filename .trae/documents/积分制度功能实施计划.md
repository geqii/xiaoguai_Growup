# 积分制度功能实施计划

## 1. Summary

* 目标：在现有错题/周计划系统中新增「积分制度」模块，支持：

  * 积分制度配置（可编辑，且可新增积分条目）

  * 日积分：每日打卡（语文/数学/英语各 1 分 + 按时早起早睡 1 分），并支持“睡前家长评判”手动加/扣分（不限范围）

  * 周积分：周末表现评定（0\~40 分，手动录入）

* 考试积分：录入分数后按规则自动计算积分；单元测验/期中考试/期末考试分别支持倍率（×1/×2/×3）

  * 其他积分事件：自由录入（可正可负）

  * 明细查询：按时间筛选、分页（20条/页）、总分汇总与分类小计

## 3. Assumptions & Decisions

* 日积分采用“每日打卡”模式：家长勾选即可（展示作息规则提醒，不要求录入实际时间）。

* 睡前家长评判：允许手动加/扣分（任意整数），并记录评语/说明。

* 明细分页默认 20 条/页。

* 考试积分自动算分，但允许在录入时手动覆盖最终积分（保留自动计算结果用于参考）。

* 语文含作文：

  * 95<=分数<100 统一按 95-99 档（+60）

  * 扣分规则按“同比例扣分”：70-80 -30；60-70 -60；<60 -120

## 4. 规则定义（一期默认内置，可在后台配置中编辑）

### 4.1 日积分（每日总计 4 分）

* 语文学习：+1

* 数学学习：+1

* 英语学习：+1

* 按时早起早睡：+1

  * 规则提醒（仅展示，不强制自动判断）：

    * 周一到周五：6:40 起床

    * 周日到周四：21:30 上床

    * 周五到周六：22:00 上床

### 4.2 周积分

* 周末完成良好：0\~40（手动输入）

### 4.3 单元测验/期中考试/期末考试（倍率）

* 倍率：

  * 单元测验：×1

  * 期中考试：×2

  * 期末考试：×3

### 4.4 单元测验/期中考试/期末考试积分规则（语文不含作文 / 数学 / 英语）

* 90<=分数<95：+25

* 95<=分数<100：+50

* 分数=100：+100

* 70<=分数<80：-25

* 60<=分数<70：-50

* 分数<60：-100

### 4.5 语文含作文积分规则

* 90<=分数<95：+30

* 95<=分数<100：+60

* 分数=100：+120

* 70<=分数<80：-30

* 60<=分数<70：-60

* 分数<60：-120

## 5. Proposed Changes

### 5.1 SQLite 表结构

* 修改文件：

  * [schema.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/db/schema.js)

* 新增表：

  1. `point_items`（积分条目/规则条目，可编辑/可新增）

     * `id TEXT PRIMARY KEY`

     * `code TEXT NOT NULL UNIQUE`（如 `daily_chinese`、`exam_math`）

     * `name TEXT NOT NULL`

     * `category TEXT NOT NULL`（`daily`/`weekly`/`exam`/`other`）

     * `kind TEXT NOT NULL`（`fixed`/`manual`/`exam`）

     * `default_points INTEGER`（fixed 用）

     * `min_points INTEGER`、`max_points INTEGER`（manual 用；允许为空表示不限）

     * `config_json TEXT`（exam 用，存档位/倍率等结构化配置，JSON 字符串）

     * `is_active INTEGER NOT NULL DEFAULT 1`

     * `created_at TEXT NOT NULL`

     * `updated_at TEXT NOT NULL`
  2. `point_events`（积分明细事件）

     * `id TEXT PRIMARY KEY`

     * `occurred_date TEXT NOT NULL`（YYYY-MM-DD，用于筛选/汇总）

     * `category TEXT NOT NULL`（冗余存，便于筛选）

     * `item_code TEXT NOT NULL`（对应 point\_items.code）

     * `points_delta INTEGER NOT NULL`（可正可负）

     * `note TEXT`（评语/说明/来源）

     * `meta_json TEXT`（JSON 字符串，存分数/科目/倍率/自动算分等）

     * `created_at TEXT NOT NULL`
  3. `daily_checkins`（每日打卡态，便于 UI 直接编辑）

     * `date TEXT PRIMARY KEY`

     * `chinese_done INTEGER NOT NULL DEFAULT 0`

     * `math_done INTEGER NOT NULL DEFAULT 0`

     * `english_done INTEGER NOT NULL DEFAULT 0`

     * `sleep_done INTEGER NOT NULL DEFAULT 0`

     * `parent_note TEXT`（睡前评语）

     * `parent_points_delta INTEGER NOT NULL DEFAULT 0`（睡前加/扣分）

     * `created_at TEXT NOT NULL`

     * `updated_at TEXT NOT NULL`

* 索引：

  * `idx_point_events_occurred_date`（occurred\_date）

  * `idx_point_events_category`（category）

  * `idx_point_events_item_code`（item\_code）

### 5.3 后端 API

* 新增文件：

  * `src/server/routes/pointsRoutes.js`

* 修改文件：

  * [app.js](file:///Users/qige/Documents/trae_projects/mistake_notebook/src/server/app.js)（挂载 `/api/points`）

* API 设计（建议）：

  * **规则管理**

    * `GET /api/points/items`：返回所有积分条目（含配置）

    * `POST /api/points/items`：新增条目（支持 fixed/manual）

    * `PUT /api/points/items/:code`：更新条目（名称/默认分/范围/启用等；exam 配置也可更新）

  * **日打卡**

    * `GET /api/points/daily?date=YYYY-MM-DD`：返回该日 checkin + 计算后的当日积分小计

    * `PUT /api/points/daily?date=YYYY-MM-DD`：更新打卡状态与睡前评语/加扣分，并同步写入 `point_events`（可用稳定 event id：`daily:<date>:<code>`）

  * **考试录入（自动算分）**

    * `POST /api/points/exams`

      * body: `{ date, subject, score, chinese_has_essay, exam_type, note?, override_points? }`

      * 返回：`{ id, calculated_points, final_points }`

* 规则：按默认档位计算基础分，再按考试类型乘倍率（单元测验×1、期中考试×2、期末考试×3）；若 override\_points 提供则 final\_points=override

  * **通用事件录入（周积分/其他积分）**

    * `POST /api/points/events`：body `{ date, item_code, points_delta, note? }`（校验范围/类型）

  * **明细与汇总**

    * `GET /api/points/events?from=YYYY-MM-DD&to=YYYY-MM-DD&page=1&pageSize=20&category?=...`

      * 返回：`{ items, page, pageSize, totalCount, summary: { totalPoints, byCategory } }`
