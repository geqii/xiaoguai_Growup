# 错题记录 Web 应用实施计划

## 1. Summary
- 目标：从零搭建一个可在局域网访问的错题记录 Web 应用，服务上海小学四年级下阶段，学科包含语文、数学、英语。
- 核心功能：
  - 快速录入：学科下拉选择、拍照上传、离线 OCR（支持手动修正与手动录入）、记录解题思路与错误原因。
  - 查看筛选：按学科浏览错题，并可按错误类型筛选。
  - 文档导出：按学科手动导出汇总 Word 文档。
- 目录要求：项目包含 `src`、`logs`、`test` 目录。

## 3. Assumptions & Decisions
- 已确认决策：
  - 技术栈：React + Node.js 全栈。
  - 部署方式：局域网多设备访问。
  - 登录：第一版无需登录。
  - OCR：本地离线 OCR，且支持手动修改识别结果与手动录入。
  - 数据库：SQLite 本地数据库。
  - 图片：保存原图到本地文件系统。
  - OCR 语言：中文 + 英文。
  - 导出：按学科汇总 Word，手动点击导出。
  - 来源信息：需要完整来源（试卷/作业名、日期、题号等）。
- 学科与错误类型策略：
  - 学科固定：`语文`、`数学`、`英语`。
  - 错误类型按学科固定：
    - 数学：`计算问题`、`概念逻辑问题`、`审题问题`
    - 语文：`基础`、`阅读理解`
    - 英语：`听力`、`辨音`、`阅读理解`、`语法`

## 4. Proposed Changes

### 4.1 项目初始化与目录结构
- 创建目录：
  - `src/client`：React 前端
  - `src/server`：Node 后端
  - `src/shared`：前后端共享常量与类型
  - `src/uploads`：错题原图存储（按学科/日期分层）
  - `src/exports`：导出的 Word 文件
  - `logs`：应用运行日志
  - `test`：测试代码
- 初始化 `package.json`，统一脚本（开发、构建、测试、导出检查）。
- 增加基础环境配置（例如端口、数据路径、OCR 配置项）。

### 4.2 后端（Node + Express）实现
- 建立后端入口与分层：
  - `src/server/app.(ts|js)`：Express 应用装配
  - `src/server/routes/*.ts`：业务路由
  - `src/server/services/*.ts`：OCR、导出、查询服务
  - `src/server/db/*`：SQLite 连接、建表、访问层
- 数据模型（SQLite）：
  - `mistakes` 表：`id`、`subject`、`error_type`、`question_text`、`analysis`、`solution_idea`、`source_name`、`source_date`、`source_question_no`、`image_path`、`created_at`、`updated_at`
  - 约束：`subject` 仅允许三门学科；`error_type` 需匹配对应学科允许集合。
- API 设计：
  - `POST /api/mistakes`：新建错题（支持上传图片或纯文本）
  - `GET /api/mistakes`：按学科、错误类型、关键词查询
  - `GET /api/mistakes/:id`：查看单条详情
  - `PUT /api/mistakes/:id`：更新错题（含 OCR 文本修订）
  - `POST /api/ocr`：对上传图片执行离线 OCR，返回可编辑文本
  - `POST /api/export/word`：按学科手动导出汇总 Word
  - `GET /api/meta/error-types`：返回学科到错误类型映射，供前端动态渲染
- OCR 服务：
  - 使用离线 OCR 引擎（中英双语）封装服务层。
  - OCR 结果包含原始识别文本，前端允许用户修正后再保存。
- Word 导出服务：
  - 读取某学科全部错题（或按时间排序）生成 `docx`。
  - 文档结构：标题、导出时间、每道错题（来源信息、图片引用或说明、题目文本、解题思路、错误原因）。
  - 文件输出到 `src/exports/<subject>/`。
- 日志：
  - 将请求日志与错误日志写入 `logs/`（分文件存储）。

### 4.3 前端（React）实现

  - `查看页`：

    - 错题列表与详情抽屉/详情面板。
